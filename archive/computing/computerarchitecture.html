Internal hardware and external components of a computer

Three-box model

Computer systems can be simply modelled using three "boxes":

  • Processor
  • Storage
  • Input and Output

Processor

The processor controls the system - it's the brain. it's Responsible for proper functioning of other parts of the system.

Modern processors contain millions of transistors. As of 2015, the highest transistor count is Intel's Xeon IvyBridge with over 4.3 billion transistors. Number of transistors on a specific area of silicon is said to double every 18-24 months. This is called Moore's Law, named after Gordon E. Moore, co-founder of Intel.

In the past, CPU was used to describe the processor plus main memory. Since the introduction of microprocessors, it now just refers to the microprocessor.

Storage

In principle, the main memory is the simplest part of the computer. In a von Neumann computer, it is used as a store for program instructions and data. Made out of memory chips. Each byte goes into a separate area of the chip called a memory location. Main memory is made up of RAM chips but can also contain ROM.

RAM

Random access memory is used both to read and to write. It's used by the processor to store data and then load it back. Ram is volatile - the data is lost when the power is turned off.

ROM

Read-only memory is not lost when power turned off - contents non-volatile. It is random access but once set up, cannot be written to. It's used to hold programs like the bootstrap program.

EEPROM

Electrically erasable programmable read-only memory does allow contents to be altered. However, takes much longer (over 100x) to write than to read. Flash memory is faster to write than other types of EEPROM because blocks written in one go rather than per byte. Widely used in commercial applications needing non-volatile storage e.g. SIM cards, flash storage for cameras, memory in MP3 players.

Bus

The "three boxes" are connected by a bus called the system bus. Can transmit a single binary word between processor and the store or I/O component.

In reality, a computer bus is a set of parallel wires that connect components of the computer to pass signals between them. The bus is split into three parts:

  • Data bus: a bidirectional bus to transport data around the computer system.
  • Address bus: a unidirectional bus to address memory and I/O locations
  • Control Bus: a bidirectional bus to transport control signals around the computer system.

Input and Output

For a computer to be useful, CPU needs to communicate with other devices, called peripherals.

A peripheral is simply a device that is not part of the CPU (processor or main memory). The system bus is not connected directly to peripherals. Connected to I/O controller.

There are I/O devices and I/O controllers. Devices include things like keyboards, magnetic disks, DVD-R drives and speakers. Controllers include the keyboard controller and magnetic disk controllers.

Peripherals aren't connected directly to the processor as each processor would have to be designed for every possible peripheral. This is why controllers are used. They convert signals from peripherals into something that can be interpreted by the processor. Some operate in one direction (e.g. keyboard controller) and others in both (e.g. floppy disk controller).

Secondary Storage

Secondary Storage is permanent storage memory. Not directly connected to processor. These include:

  • Magnetic hard disk
  • CD-ROM,CD-R,CD-RW
  • DVD-ROM,DVD-R,DVD-RW
  • Memory stick

Functional characteristics of a processor

Processor performs operations on data by executing program instructions. A program changes the contents of main memory by sending single words over the system bus. Program instructions and data are fetched from main memory and results are written to main memory. The processor uses numeric codes on the address bus to specify locations in main memory to choose.

Addressable memory

The processor must be able to distinguish parts of main memory. This is why each location in memory has a unique main memory address. A program requiring data from memory puts an address on the address bus. The data from that location is then sent along the data bus. All three buses are required when the processor accesses an area of main memory.

Stored program concept

The stored program concept was proposed by John von Neumann and Alan Turing separately in 1945. It said:

  • A program must be present in main memory to be executed.
  • Machine code instructions are fetched, one at a time, from main memory, and executed sequentially by the processor.

Processor instructed to perform arithmetic or logic functions (e.g. AND, OR, ADD, SUBTRACT). Instructions are represented in machine code, a numeric instruction set, specific to the processor. Since each instruction is a single number, each memory location can hold one instruction.

Types of stored program computer: von Neumann vs Harvard

In a von Neumann stored program system, data and instructions are stored in the same memory so instructions cannot be fetched at the same time as data is read or written. However, in a Harvard stored program computer, there is a separate memory and buses for instructions. This means that instructions can be fetched in parallel with data read and write, increasing the overall processing speed of the system.

Microcontroller

A microcontroller is an entire computer (including processor, memory and I/O) on a single chip. Mostly used as embedded computers, where a cheap and compact programmable system is required. An embedded computer is a special purpose system inside another device. Examples are GPS systems, iPods and routers.

Structure and role of the processor

A processor comprises the following components:

  • Program control unit fetches program instructions from memory, decodes and executes them one at a time.
  • Arithmetic and logic unit performs arithmetic and logical operations on inputted data. Examples include addition, subtraction, AND, OR, XOR. Also can perform shifts.
  • Registers are quick memory locations contained in the processor. Faster than main memory since no external buses required.
  • Internal clock is derived directly or indirectly from the system clock.
  • Internal buses are buses to link the ALU, control unit and registers.
  • Logic gates is used for flow control.

General-Purpose and dedicated registers

Registers are very fast memory locations inside the processor. General-purpose registers (R0,R1 etc.) are available for the programmer's use to store temporary data. They are not assigned a specific role by the processor designer. Accessed via instructions like LOAD, STORE and ADD.

NB: R1 is the register store with address 1. Not the same as a memory location with address 1.

Dedicated registers are those with a specific role, chosen by the processor designer. They have names like SP, PC, SR and ACC. Programmers can read and change some, but not all of these. Here are some dedicated registers:

  • Program counter (PC):points to next instruction to be fetched and executed.
  • Status register (SR): holds condition codes to show the outcome of operations. Example: an arithmetic operation could produce a positive, negative, zero or overflow result. The flag is set accordingly.
  • Accumulator (ACC): holds the result of the current set of calculations. E.g. ADD #36 adds the number 36 to what is currently in the ACC and then stores it again.
  • Current instruction register (CIR): holds the current instruction, while it is decoded and executed.
  • Memory address register (MAR): holds the address of the memory location being accessed by the processor.
  • Memory buffer register (MBR): holds the data being transferred to or from the main memory location being accessed by the processor.

System clock and clock speed

Every computer system has a quartz-controlled oscillator which sends out a timing signal at a fixed rate. All other timings are derived from this system clock. Controls how fast instructions are executed and synchronises various parts of the computer. Clock rates are measured in megahertz (MHz) or gigahertz (GHz).

Processors are designed to perform instructions at a specified frequency. This is called the clock speed. Each instruction is executed by the processor in a fixed number of clock cycles. Some require more cycles than others. Clock cycles are also required for operation of the system bus to transfer binary words.

Computer clock speeds have been doubling roughly every year.

Word length

Computers work with binary words of specific lengths. Each wire in a bus corresponds to a bit in the maximum word length of the system. The word length is the number of bits in the word.

Effect of clock speed, word length and bus width on performance

For a processor, performance is how fast a specified task can be completed. The unit of performance is float-point operations per second (FLOPS).

Increasing clock speed

A processor with double the clock speed should execute the same machine code program twice as fast, as long as all other factors are constant. This is because each cycle is twice as fast so each instruction will take half the time to complete.

Limits on clock speed

A limit must be set on clock speed because otherwise, the heat generated in the chip cannot be dissipated fast enough, leading to chip damage. This problem is exacerbated when transistors are packed closer together, as the trend of Moore's law shows occurs more and more.

Multicore microprocessors

What is now done is to put more than one processor onto the chip. These cores operate at a lower clock speed than a single core and so the temperature problem is... dissipated. This also allows multiple tasks to be run at the same time, improving performance significantly.

Increasing word length

The word length of registers and buses affects the processor's speed performance. The word length denotes the largest number which can be computed by the system. Anything larger must be split into multiple words. This is time consuming and affects performance negatively. Typical word lengths are 32 or 64 bits. Also, since words are used for addressing, it limits the amount of memory which can be used in a system.

Increasing bus width

The worst time penalty occurs in the system bus between processor and main memory. If the CIR is not large enough to hold a complete instruction, the system bus must be used to transfer multiple words. This can slow down the processor as the system bus is a bottleneck. A wider system bus allows more data to be transferred per second and a wider address bus increases the maximum possible memory which can be installed into the computer.

Revision points:

  • The three-box model consists of a processor, main memory and I/O connected by a system bus.
  • A system bus is made up of a data bus, an address bus and a control bus. A system bus is a shared bus.
  • An I/O controller is the interface between the system bus and an I/O device. It contains I/O ports
  • Secondary storage is permanent storage, e.g. a magnetic disk
  • Peripherals are not part of the processor and main memory.
  • Memory Locations are uniquely identified by address codes.
  • In the stored program concept, a program must be resident in main memory to be executed.
  • A program is processed by fetching machine code instructions in sequence from main memory then decoding and executing them, one at a time, in the processor.
  • A processor consists of an ALU, a control unit and instruction decoder, registers, an internal clock, internal buses and logic gates.
  • A system clock provides timing signals so the system can run properly.
  • Increasing the clock frequency or the word length increases the processor speed of operation.
  • Increasing the address bus width allows more memory to be addressed. Increasing the data bus width increases the speed of operation.
