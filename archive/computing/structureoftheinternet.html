The internet is a network of networks. It began in the 1960s, along with sex and drugs. All the computers on the internet are part of the global namespace governed by TCP and IP.

Packets

Whenever some information needs to be sent from one place to anther, it must be split up into packets. Usually, these packets are around 576 bytes.

Each packet contains (as well as the data it is carrying) the IP address and port of the place it is headed and the place it came from, so that any data can be returned.

The packets are sent by different routes to the destination, to make it less likely that they are all lost in transmission if one route fails.

The end-to-end principle states that the two end computers should be in control of establishing, maintaining and supervising the communication and that the internet's only role is to move packets between them.

The client-server model is the scenario when a process on the client machine initates a connection to a server and sends a request, and then the server responds to that request.

TCP IP Stack

All the information that travels through the internet passes up and down the network stack.

  1. Application
  2. Transport
  3. Network
  4. Link

Application Layer

The application layer deals with the protocols that are specific to each type of traffic, e.g. FTP for file transfers, HTTP(S) for the web, SSH for remote shells etc.

Transport Layer

This is the layer that TCP (and UDP) operate on. They follow the end-to-end principle. TCP enables a reliable flow of data from one computer to another (making sure that all the data gets to the destination) and deals with splitting the data into packets so the applications do not have to.

TCP is resposible for error checking and resending of any dropped packets as well as maintaining the connection so the applications do not have to.

It adds the source and destination TCP port onto the packet, so one device can maintain many connections simultaneously on different port numbers.

Network Layer

The network layer is responsible for addressing the packets so the routers know where to send them, as well as a return address so the routers know where to send the response. This is in the form of an IP address.

The combination of the IP address and port number is called a socket.

Link Layer

The link layer is the last layer in the stack, dealing with the hardware. It adds source and destination hardware (MAC) addresses to the packets, and then dispatches them onto the network (usually down an Ethernet cable or over a WiFi network).

Encapsulation

The result of passing packets up and down the stack is that the data is "encapsulated" several times, with the data at the middle, the TCP ports, then the IP addresses, and then the hardware addresses.

Hardware

There are only a few different types of hardware on the internet.

Routers

Routers receive packets and forward them on according to their destination IP addresses. A router is in charge of allocating IP addresses to devices that are underneath it in the network tree. Sometimes this means passing it up the chain to another router, and sometimes it can send it directly to a host that is connected to it. It uses subnet masks to figure out whether or not it is in charge of the destination host.

Gateways

Gateways are used to transport traffic from one link-layer type to another (e.g. from WiFi to Ethernet, Ethernet to Fibre etc). Any router that includes WiFi is also a gateway.

Common Protocols

Protocols are a set of agreed rules governing communication between two systems, usually specialised to one use.

  • HTTP (Port 80): Insecure Web
  • HTTPS (Port 443): Secure Web
  • FTP (Port 20 & 21): Insecure File Transfers
  • SSH (Port 22): Secure Remote Console
  • Telnet (Port 23): Old, bad remote console

Ports 0-2013 are ports which are universally used by servers for different purposes. Client port numbers, on the other hand, are 1024 upwards and are used by client processes and are temporary.

Network Topology

Each company/school/home nowadays has one "main" router with a connection to the internet. All devices on the LAN (local area network) of the private routers are "hidden" behind that router, such that the outside world cannot differentiate between them as they only have one global IP address. These private routers are then connected to local routers, regional routers and finally national routers which are also connected to the international network.

IP addresses are allocated by region, so you can tell the location of a host by its IP address. Local IP addresses on the router's (private) network are in the range of 10.*.*.* , 192.168.*.* or 172.16.*.* to 172.31.*.*.

Domain Names & URIs

Domain names are essentially human-memorable aliases for IP addresses, e.g. stpaulsschool.org.uk. When a user wants to send data to that domain, their computer first translates that into an IP address using a DNS server, which keeps a list of which domains correspond to which IP addresses.

A domain name can have multiple IP addresses associated with it, for instance one for each type of server. A fully qualified domain name (FQDN) is a domain name that refers to a single IP address (e.g. mail1.stpaulsschool.org.uk).

Domain names are made up of levels of domain. For instance, mail.stpaulsschool.org.uk has these levels:

  • mail: the subdomain
  • stpaulsschool: the domain
  • org: the second level domain
  • uk: the first level domain

URIs

A URI is a uniform resource identifier and it specifies how to access a resource of the Internet.

A URL on the other hand is a uniform resource locator or a URI that identifies a resource by its network location.

Take this URI for example: https://www.stpaulsschool.org.uk:2000/stuff.html

The first part is the protocol to be used, in this case HTTPS. Next is the domain name of the webserver that we want. Everything after the colon is the port to be used. If it is omitted, the default port for that protocol is used (443 in this case). Everything after the port number is the location of the document to be accessed on the server.

The World Wide Web

The web is a system interlinked hypertext documents accessed via the Internet. They can be accessed using web browsers or anything else that can speak HTTP(S).

Intranets, on the other hand, are private computer networks that are owned by an organisation to share information with its employees using Internet protocols.
