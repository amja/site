System Software

Operating Systems

Operating Systems are the interface between the computer and the user. They allow the hardware to be used without accessing it directly. In order for this to happen the Operating System creates a 'virtual machine', which hides the complexity of the hardware from the user with an alternate interface. Operating Systems create the platform from which other pieces of software run, and thus most applications are written for a specific Operating System: for instance, a program written to run on *Non-specific Brand* PCs using Linux might not run on *Non-specific Brand* PCs using GNU

Library Programs

Library Programs are collections of compiled routines which will be used by multiple other programs: a good example are print routines, which will be used by both word processors and spreadsheet programs.

Utility Programs

Utility programs perform specific tasks, like file compression or disk formatting. Normally some are bundled with the Operating System, while others can be downloaded.

Language Translators

These are not translators between human languages, but between types of code language. Computers can only run Machine Code, and if a program is in a higher-level programming language a translator is necessary in order to exxecute it.

Assemblers

An assembler is a program that translates assembly language into machine code. Each expression in turn is converted to the binary equivalent.

Compilers

A compiler converts a whole program in a high-level code directly into object code. The resulting translation can then be distributed without the need for real-time translation

Interpreters

An interpreter analyses and executes each line of a high-level language program individually in sequence. This means the program must be interpreted every time it is run. The program can be executed much faster than compiling, at least initially, but must be interpreted every time, and thus is slower in the long term than a compiled program, and also allows for easier copying of source code.

Application Software

Bespoke Software

Bespoke software is custom software developed for a specific customer, who pays for all the development costs. A good example is Department Store stock and department organisation software

General-Purpose Software

General-purpose software can be used for a variety of purposes. Good examples are word processors or graphing software.

Special-Purpose Software

Special-purpose software is designed for a specific task, but not a single consumer. Good examples are Air Traffic Control software that might be shared between multiple airports and web browsers.

Generations of Programming Language

First Generation Languages

Machine Code - the only language a computer can execute directly. Unique to a family of processors.

Second Generation Languages

Assembly Code - Uses same structure as Machine Code, but uses mnemonics rather than numbers.

Third Generation Languages

Imperative languages - must be executed in a programmer-defined sequence. Also known as 'procedural languages'.

Fourth Generation Languages

Declarative Programming Languages - define what needs to be done, rather than how to do it. Used to write or declare facts that the computer interprets and executes.
